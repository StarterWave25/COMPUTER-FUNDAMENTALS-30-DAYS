# The Last Laptop Problem

### Scenario

Tomorrow morning is your project demo.

Your team has only **one working laptop** that can be used for development and testing.

Team Members:

| Member | Task                 | Time Needed |
| ------ | -------------------- | ----------- |
| Prudvi | Fix Login Bug        | 20 min      |
| Purush | Build Dashboard UI   | 3 hrs       |
| Reddy  | Prepare Presentation | 1 hr        |
| Jagan  | Record Demo Video    | 30 min      |
| Ganesh | Fix Payment Bug      | 15 min      |

### Rules

* Only one person can use the laptop at a time.
* The demo starts tomorrow at 9:00 AM. The time is 5:00PM.
* Everyone can work upto 9:30PM i.e, 4-1/2 Hours but to complete it needs 5 1/2 hours.
* All tasks are important.
* The team must agree on a schedule.

### Challenge

As a team, create the order in which the laptop should be used.

Do not think about operating systems.

Think only about finishing the demo successfully.

---

# Twist 1 (Reveal after they finalize the first schedule)

After 10 minutes, the faculty coordinator sends a message:

> "I need the presentation slides within the next 30 minutes for review."

### New Question

Should Reddy move ahead of others?

If yes:

* Who gets delayed?
* Why?

If no:

* What risks are you accepting?

The whole team must agree.

---

# Twist 2 (Reveal after they solve Twist 1)

Ganesh discovers something serious:

> "The payment bug is causing every transaction to fail. If it's not fixed, the demo won't work."

### New Question

Should Ganesh immediately get the laptop?

If yes:

* Is it fair to others already waiting?
* Should urgent work always jump the queue?

If no:

* What happens if the demo breaks?

The whole team must agree.

---

# Twist 3 (Reveal after they solve Twist 2)

Purush has already been working on the Dashboard UI for 45 minutes.

Suddenly:

* Prudvi finds another login issue.
* Jagan realizes the demo video needs one quick retake.

Both need only 5 minutes.

### New Question

Should Purush continue until he finishes?

Or should he temporarily give up the laptop so smaller tasks can complete first?

The team must decide:

* What is the best use of the laptop?
* What is the fairest approach?
* What maximizes the chances of a successful demo?
